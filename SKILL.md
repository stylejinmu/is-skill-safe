# Is Skill Safe

This skill is designed to audit other Manus skills. It analyzes the `SKILL.md` file and any associated scripts to explain the skill's purpose, identify potential security risks, and provide an assessment of whether it is safe to use.

## How to Use

When you need to audit a skill, follow these steps:

1. **Run the Audit Script**: Execute the provided Python script against the target skill directory.
2. **Analyze the Output**: Review the output generated by the script, which highlights potential risks in both the `SKILL.md` file and any associated scripts.
3. **Provide a Summary**: Based on the script's output and your own understanding of the skill's code, provide a comprehensive summary to the user.

### Step 1: Run the Audit Script

Use the `shell` tool to run the `audit_skill.py` script. Pass the absolute path to the target skill directory as an argument.

```bash
python /home/ubuntu/skills/is-skill-safe/scripts/audit_skill.py <path_to_target_skill>
```

For example, to audit a skill named `example-skill`:

```bash
python /home/ubuntu/skills/is-skill-safe/scripts/audit_skill.py /home/ubuntu/skills/example-skill
```

### Step 2: Analyze the Output

The script will output:
- The name and description of the skill from its frontmatter.
- Any potentially risky patterns found in the `SKILL.md` file (e.g., `rm -rf`, `sudo`, `eval`).
- An analysis of each script in the `scripts/` directory, highlighting potentially dangerous operations (e.g., `os.system`, `subprocess.run`, file write operations, HTTP requests).

### Step 3: Provide a Summary

After running the script, synthesize the findings into a clear report for the user. Your report should include:

1. **Explanation of Purpose**: What does the skill do based on its description and code?
2. **Security Risks Identified**: List any potential risks found by the script or through your own manual review of the code. Explain *why* these might be risky in the context of the skill.
3. **Safety Assessment**: Conclude whether the skill is generally safe to use. If there are risks, advise the user on how to mitigate them or whether they should avoid using the skill altogether.

## Important Considerations

- **Static Analysis Limitations**: The `audit_skill.py` script performs basic static analysis using regular expressions. It may produce false positives (flagging safe code as risky) or false negatives (missing obfuscated or complex malicious code).
- **Manual Review**: Always supplement the script's findings with your own understanding of the code. If a script uses `subprocess.run`, check *what* command it is running. If it writes to a file, check *where* and *what* it is writing.
- **Context Matters**: Some "risky" operations (like writing to a file or making an HTTP request) are perfectly normal and necessary for certain skills. Evaluate the risk in the context of the skill's stated purpose.
